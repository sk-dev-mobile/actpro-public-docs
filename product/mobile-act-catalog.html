Каталог и структура акта

Сценарий загрузки каталога для составления акта на устройстве и построения локального дерева помещение → группа → дефект.

Что делает сценарий

1. Загружает GET /api/v1/defects-reference/catalog.

2. Собирает справочник комнат, групп, дефектов и стандартов.

3. Даёт mobile-клиенту все идентификаторы для последующего POST /api/v1/acts.

Prerequisites

Нужен валидный access token эксперта, менеджера или администратора.

Шаги

1. Один раз скачать каталог после авторизации.

2. Положить его в локальный storage как offline snapshot.

3. В UI подставлять названия комнат, групп и дефектов по глобальным массивам rooms, groups и defects.

4. Для конкретного стандарта использовать standard_room_id и local_groups как рабочую структуру выбора дефекта.

Endpoints

GET /api/v1/defects-reference/catalog

Возвращает глобальные справочники и стандарты с помещениями, отделками, группами, override-основаниями и локальными группами.

Request

curl -sS 'https://api.lgtime.ru/api/v1/defects-reference/catalog' \ -H 'Authorization: Bearer <access_token>'

Response

{ "version": "2026-04-23T12:00:00Z", "rooms": [ { "id": 5, "name": "Спальня" } ], "groups": [ { "id": 12, "name": "Стены", "defect_ids": [345, 346] } ], "defects": [ { "id": 345, "document_name": "Трещина", "app_name": "Трещина", "default_normative": "СП ..." } ], "standards": [ { "id": 3, "name": "Базовый стандарт", "is_archived": false, "rooms": [ { "standard_room_id": 101, "room_id": 5, "sort_order": 10, "finishes": [ { "finish_type": "finish", "group_ids": [12], "overrides": [], "local_groups": [ { "group_id": 12, "defect_ids": [345] } ] } ] } ] } ] }

Ошибки

401 при отсутствии или истечении access token.

403 если роль пользователя не имеет доступа к каталогу.

Edge cases

standard_room_id — это не тип комнаты, а конкретная строка помещения внутри стандарта.

local_groups содержит только group_id и defect_ids. Названия надо брать из верхнеуровневых groups и defects.

overrides задаёт локальную нормативку на уровне пары group_id + defect_id.

Что хранить локально

Полный ответ каталога и дату последней синхронизации. Этого достаточно для offline выбора дефектов и подготовки payload акта.

Текущие ограничения

Для нового mobile-клиента предпочтителен разбор через local_groups. Поле group_ids стоит считать совместимым списком привязанных групп, а не полным составом локальной группы.
