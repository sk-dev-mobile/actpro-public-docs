Доступ и роли

Схема входа и прав в АктПро.

Цель: простой вход и понятное разделение прав.

Основа

В системе есть Организация.

Пользователь работает внутри своей организации и получает роль.

По правилам продукта пользователь относится к одной организации.

При необходимости это правило можно расширить позже.

Роли

Администратор

Полный доступ внутри организации.

Менеджер

Настраивает справочники, создает экспертов, выдает доступ, редактирует отчеты и заключения.

Эксперт

Работает в приложении и заполняет акты.

Дальше: позже для эксперта можно открыть ограниченный доступ через браузер.

Вход

Администратор и менеджер

Вход: почта и пароль.

Эксперт

Вход: уникальный четырехзначный код.

После входа приложение сохраняет долгую сессию.

Создание организации

Организация клиента создается вручную внутренним оператором системы.

Первый администратор также создается вручную.

Самостоятельной регистрации организации в первой версии нет.

Почему вход разный

  • администратор и менеджер работают в браузере;
  • эксперт работает в поле и должен входить быстро;
  • все остаются пользователями одной системы.

Схема

flowchart TD organization[Организация] user[Пользователь] membership[Роль в организации] adminManager[Администратор или менеджер] expert[Эксперт] browserAccess[Вход по почте] codeAccess[Вход по коду] app[Приложение] longSession[Долгая сессия] organization --> membership user --> membership membership --> adminManager membership --> expert adminManager --> browserAccess expert --> codeAccess codeAccess --> app app --> longSession

Важно: роль хранится внутри связи пользователя с организацией.

Выдача кода эксперту

flowchart LR manager[Менеджер] createExpert[Создает эксперта] generateCode[Система создает код] showCode[Показывает код один раз] expert[Эксперт] app[Приложение] session[Долгая сессия] manager --> createExpert createExpert --> generateCode generateCode --> showCode showCode --> expert expert --> app app --> session
  • код создает система;
  • код выдает или перевыпускает менеджер;
  • менеджер видит код только в момент создания или перевыпуска;
  • позже старый код больше не показывается;
  • если нужен новый доступ, менеджер выпускает новый код.

Границы первой версии

  • самостоятельной регистрации нет;
  • первая организация и первый администратор создаются управляемо;
  • эксперт работает только в приложении;
  • пользователь работает только в одной организации.

Защита

Четырехзначный код удобен и требует дополнительных мер защиты.

Для защиты доступа в системе предусмотрены базовые ограничения и контроль действий.

  • ограничение числа попыток входа;
  • временная блокировка после ошибок;
  • хранение кода только в защищенном виде;
  • сброс кода менеджером;
  • отзыв активных сессий;
  • журнал входов и ошибок.
