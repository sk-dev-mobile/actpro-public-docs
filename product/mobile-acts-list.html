Просмотр актов

Сценарий просмотра ранее созданных актов: список, детали и получение pre-signed ссылки на PDF.

Что делает сценарий

1. Получает список актов через GET /api/v1/acts.

2. Открывает детали конкретного акта через GET /api/v1/acts/{act_id}.

3. Запрашивает ссылку на PDF через GET /api/v1/acts/{act_id}/pdf.

Prerequisites

Нужен access token пользователя с ролью expert, manager или admin.

Шаги

1. Запросить список с limit и offset.

2. Прочитать заголовок X-Total-Count для пагинации.

3. При открытии карточки вызвать GET /acts/{act_id}.

4. Для просмотра PDF вызвать GET /acts/{act_id}/pdf?mode=inline.

Endpoints

GET /api/v1/acts

Query params: limit, offset, q, act_type, client_id, опционально expert_id.

GET /api/v1/acts/{act_id}

Возвращает полный акт, список дефектов и состояние PDF.

GET /api/v1/acts/{act_id}/pdf

Возвращает pre-signed URL и время его истечения.

Request

curl -sS 'https://api.lgtime.ru/api/v1/acts?limit=20&offset=0&act_type=expertise' \ -H 'Authorization: Bearer <access_token>'
curl -sS 'https://api.lgtime.ru/api/v1/acts/42' \ -H 'Authorization: Bearer <access_token>'
curl -sS 'https://api.lgtime.ru/api/v1/acts/42/pdf?mode=inline' \ -H 'Authorization: Bearer <access_token>'

Ошибки

401 при отсутствии access token.

403 если роль не имеет доступа к актам.

404 если акт не найден или недоступен в организации.

409 на PDF route, если PDF ещё не готов или с ним есть конфликт состояния.

Edge cases

Пагинация у списка основана на limit, offset и заголовке X-Total-Count.

Поля expert.display_name, room_name, group_name и defect_name уже обогащены backend и не требуют отдельного lookup.

Что хранить локально

Фильтры списка, последний offset, заголовок X-Total-Count и при необходимости краткий кеш последних открытых деталей.

Текущие ограничения

Отдельного mobile-specific list endpoint нет. Текущий flow — это обычные list/detail routes с пагинацией через limit и offset, без cursor API.
